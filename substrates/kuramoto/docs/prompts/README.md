# LLM Development Prompts

This directory contains LLM system prompts and agent configurations that were used during the development of TradePulse. These files are **not part of the runtime system** and are retained for historical reference only.

## Contents

- **system_prompt_principal_architect.md** - System prompt defining architectural methodology and frameworks used during TradePulse design
- **doc_pr_copilot_guide.md** - Configuration guide for documentation automation agents

## Purpose

These prompts document the development methodology and AI-assisted workflows used to build TradePulse architecture and documentation. They provide transparency into the design process and may be useful for:

- Understanding architectural decision-making frameworks
- Adapting similar methodologies for other projects
- Historical reference of development practices

## Note

These files are **internal development artifacts** and do not affect the operation of TradePulse. The actual system architecture is documented in:

- [docs/ARCHITECTURE.md](../ARCHITECTURE.md)
- [docs/CONCEPTUAL_ARCHITECTURE_UA.md](../CONCEPTUAL_ARCHITECTURE_UA.md)
- Architecture Decision Records in [docs/adr/](../adr/)
