# Archive Directory

This directory contains legacy documents, drafts, and historical artifacts that are no longer actively used by the current system but are preserved for historical context and reference.

## Contents

All files in this directory should be prefixed with `LEGACY_YYYYMMDD_` to indicate:
- That they are not current documentation
- The approximate date they became legacy (or were moved here)
- Their original context

## Purpose

These files are kept because they:
- Provide historical context about project evolution
- Document completed tasks and implementation decisions
- May contain useful insights for future work
- Are referenced in git history or old discussions

## What NOT to Look Here For

For current, active documentation, see:
- [README.md](../../README.md) - Project overview
- [docs/](../) - Current documentation directory
- [ARCHITECTURE.md](../ARCHITECTURE.md) - System architecture
- [TESTING.md](../../TESTING.md) - Testing guide
- [SECURITY.md](../../SECURITY.md) - Security documentation

## Retention Policy

Legacy documents are retained indefinitely unless they:
- Are superseded by identical content in active docs
- Contain outdated or misleading information that could cause confusion
- Are empty, corrupted, or unreadable

When in doubt, keep the file with clear LEGACY marking.
