# TradePulse Calibration Guide

## Overview

This guide provides comprehensive instructions for calibrating accuracy, thresholds, and sensitivity parameters across TradePulse controllers and modules. Proper calibration is essential for optimal trading performance under different market conditions.

## Introduction

TradePulse employs multiple controllers and modules that work together to make trading decisions:

- **NAK Controller**: Manages neuro-energetic state and trading limits
- **Dopamine Controller**: Handles reward prediction and action selection
- **Serotonin Controller**: Models chronic stress and produces hold decisions
- **Risk Engine**: Enforces position limits and loss thresholds
- **Regime Adaptive Guard**: Dynamically scales exposure based on volatility
- **Rate Limiter**: Controls API and execution rate limits
- **GABA Controller**: Provides inhibitory control against impulsive actions
- **Desensitization**: Manages receptor adaptation under repeated stimulation

Each controller has configurable parameters that affect its behavior. This guide explains how to calibrate these parameters for your specific trading scenario.

For detailed parameter ranges and defaults, see [CALIBRATION_PARAMETER_REFERENCE.md](CALIBRATION_PARAMETER_REFERENCE.md).

## Key Concepts

### Accuracy
The precision and correctness of predictions and signals generated by controllers. Higher accuracy reduces false signals but may miss opportunities.

### Thresholds
Boundary values that trigger state changes or actions. Examples:
- Engagement Index (EI) thresholds determine when trading is suspended
- Volatility thresholds trigger risk mode changes
- Loss thresholds activate kill switches

### Sensitivity
How quickly controllers respond to market changes. Higher sensitivity means faster reactions but more noise; lower sensitivity provides stability but slower adaptation.

## Controllers and Modules

### NAK Controller

The NAK (Neuro-Arousal-Ketosis) Controller manages energy, load, and engagement to determine trading activity levels.

**Key Parameters:**
- `EI_low`, `EI_high`: Engagement index band (normal operation range)
- `EI_crit`: Critical engagement threshold (below this = suspension)
- `vol_amber`, `vol_red`: Volatility thresholds for risk mode transitions
- `dd_amber`, `dd_red`: Drawdown thresholds for risk mode transitions
- `delta_r_limit`: Maximum rate of change in risk factor (rate limiting)
- `risk_mult`: Risk multipliers per mode (GREEN/AMBER/RED)
- `activity_mult`: Activity multipliers per mode

**Configuration Location:** `nak_controller/conf/nak.yaml`

### Dopamine Controller

The Dopamine Controller implements reward prediction error (RPE) and action selection with exploration/exploitation balance.

**Key Parameters:**
- `learning_rate_v`: Speed of value function learning
- `burst_factor`: Amplification of phasic dopamine responses
- `base_temperature`: Exploration temperature (higher = more exploration)
- `invigoration_threshold`: Threshold for GO signal activation
- `no_go_threshold`: Threshold for NO-GO inhibition
- `discount_gamma`: Future reward discount factor

**Configuration Location:** `config/dopamine.yaml` or `config/profiles/*.yaml`

### Risk Engine

The Risk Engine enforces hard limits to protect capital.

**Key Parameters:**
- `max_position_size_default`: Maximum position size per symbol
- `max_daily_loss`: Absolute daily loss limit
- `max_daily_loss_percent`: Daily loss limit as percentage
- `max_leverage`: Maximum portfolio leverage
- `kill_switch_loss_threshold`: Loss that triggers emergency stop

**Configuration Location:** `configs/risk_engine.yaml`

## Calibration Profiles

TradePulse provides three pre-configured calibration profiles optimized for different market conditions and risk preferences:

### Conservative Profile
- **Use Case**: Low volatility markets, risk-averse trading, account preservation
- **Characteristics**:
  - Tight thresholds (smaller operating bands)
  - Lower sensitivity (slower reactions)
  - Stricter gating (easier to block trades)
  - Lower exploration (more predictable behavior)

### Balanced Profile (Default)
- **Use Case**: Normal market conditions, moderate risk tolerance
- **Characteristics**:
  - Standard thresholds
  - Moderate sensitivity
  - Balanced gating
  - Moderate exploration

### Aggressive Profile
- **Use Case**: High opportunity markets, higher risk tolerance, active trading
- **Characteristics**:
  - Loose thresholds (wider operating bands)
  - Higher sensitivity (faster reactions)
  - Permissive gating (harder to block trades)
  - Higher exploration (more opportunistic)

## Calibration Process

### Using the Calibration Utility

TradePulse provides a command-line utility for easy calibration:

```bash
# List available profiles
python scripts/calibrate_controllers.py --list-profiles

# Apply a profile to NAK controller
python scripts/calibrate_controllers.py --controller nak --profile balanced

# Apply a profile to Dopamine controller
python scripts/calibrate_controllers.py --controller dopamine --profile aggressive

# Apply a profile to Serotonin controller
python scripts/calibrate_controllers.py --controller serotonin --profile conservative

# Apply a profile to Risk Engine
python scripts/calibrate_controllers.py --controller risk_engine --profile balanced

# Apply a profile to Regime Adaptive Guard
python scripts/calibrate_controllers.py --controller regime_adaptive --profile aggressive

# Validate an existing configuration
python scripts/calibrate_controllers.py --validate conf/nak/default.yaml

# Apply with custom output path
python scripts/calibrate_controllers.py --controller nak --profile conservative --output conf/nak/custom.yaml
```

**Supported Controllers:**
- `nak` - NAK (Neuro-Arousal-Ketosis) Controller
- `dopamine` - Dopamine Controller
- `serotonin` - Serotonin Controller
- `risk_engine` - Risk Engine
- `regime_adaptive` - Regime Adaptive Exposure Guard

**Makefile Shortcuts:**
```bash
# Using make for convenience
make calibrate-list           # List all profiles
make calibrate-validate       # Validate current configs
make calibrate-conservative   # Apply conservative profile (all controllers)
make calibrate-balanced       # Apply balanced profile (all controllers)
make calibrate-aggressive     # Apply aggressive profile (all controllers)
```

### Manual Calibration Steps

1. **Identify Target Market Conditions**
   - Analyze historical volatility, liquidity, and regime characteristics
   - Determine your risk tolerance and performance objectives

2. **Select Initial Profile**
   - Start with the closest pre-configured profile
   - Conservative for low-volatility, risk-averse scenarios
   - Balanced for most situations
   - Aggressive for high-opportunity environments

3. **Test in Backtest Mode**
   ```bash
   # Run backtest with calibrated config
   python scripts/run_backtest.py --config conf/nak/balanced.yaml --data historical_data.csv
   ```

4. **Analyze Results**
   - Review Sharpe ratio, max drawdown, win rate
   - Check engagement index stability
   - Assess mode transition frequency

5. **Iterate and Refine**
   - Adjust parameters based on performance
   - Re-validate after each change
   - Test on different time periods

6. **Validate Configuration**
   ```bash
   python scripts/calibrate_controllers.py --validate your_config.yaml
   ```

7. **Deploy to Paper Trading**
   - Test with real market data but simulated execution
   - Monitor for at least 2 weeks before live trading

### Validation Checklist

Before deploying a calibrated configuration:

- [ ] All validation checks pass
- [ ] Thresholds form valid relationships (e.g., `EI_low < EI_high`)
- [ ] Parameters are within safe bounds
- [ ] Backtest performance meets targets
- [ ] Paper trading shows stable behavior
- [ ] Risk limits are appropriate for account size
- [ ] Mode transitions occur at expected volatility/drawdown levels

## Parameter Reference

### NAK Controller Parameters

#### Engagement Index Thresholds

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `EI_low` | 0.0-1.0 | 0.35 | Lower bound of normal engagement band |
| `EI_high` | 0.0-1.0 | 0.65 | Upper bound of normal engagement band |
| `EI_crit` | 0.0-1.0 | 0.15 | Critical threshold (below = suspend) |
| `EI_hysteresis` | 0.0-0.2 | 0.05 | Hysteresis for unsuspend threshold |

**Tuning Guidance:**
- Wider bands (larger `EI_high - EI_low`) = more stable operation
- Lower `EI_crit` = more aggressive (harder to suspend)
- Higher hysteresis = more stable (less flip-flopping)

#### Volatility and Drawdown Thresholds

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `vol_amber` | 0.0-2.0 | 0.70 | Volatility threshold for AMBER mode |
| `vol_red` | 0.0-2.0 | 0.90 | Volatility threshold for RED mode |
| `dd_amber` | 0.0-1.0 | 0.40 | Drawdown threshold for AMBER mode |
| `dd_red` | 0.0-1.0 | 0.70 | Drawdown threshold for RED mode |

**Tuning Guidance:**
- Lower thresholds = more conservative (enter protective modes earlier)
- Larger gaps between AMBER and RED = smoother transitions
- Match to your historical volatility and drawdown tolerance

#### Risk and Activity Multipliers

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `risk_mult.GREEN` | 0.0-2.0 | 1.00 | Risk multiplier in normal mode |
| `risk_mult.AMBER` | 0.0-2.0 | 0.65 | Risk multiplier in elevated risk mode |
| `risk_mult.RED` | 0.0-2.0 | 0.00 | Risk multiplier in high risk mode (usually 0) |
| `activity_mult.GREEN` | 0.0-2.0 | 1.20 | Activity multiplier in normal mode |
| `activity_mult.AMBER` | 0.0-2.0 | 0.90 | Activity multiplier in elevated risk mode |
| `activity_mult.RED` | 0.0-2.0 | 0.60 | Activity multiplier in high risk mode |

**Tuning Guidance:**
- `GREEN` multipliers > 1.0 encourage activity in good conditions
- `AMBER` multipliers < 1.0 reduce exposure in moderate stress
- `RED` typically suspends all activity (`risk_mult.RED = 0`)

### Dopamine Controller Parameters

#### Learning and Adaptation

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `learning_rate_v` | 0.001-0.5 | 0.10 | Value function learning rate |
| `discount_gamma` | 0.90-0.999 | 0.98 | Future reward discount factor |
| `burst_factor` | 1.0-5.0 | 2.5 | Phasic dopamine amplification |

**Tuning Guidance:**
- Higher `learning_rate_v` = faster adaptation but less stability
- Higher `discount_gamma` = more long-term oriented
- Higher `burst_factor` = stronger reactions to unexpected rewards

#### Exploration and Temperature

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `base_temperature` | 0.1-3.0 | 1.0 | Exploration temperature |
| `min_temperature` | 0.01-0.5 | 0.05 | Minimum temperature floor |
| `neg_rpe_temp_gain` | 0.0-2.0 | 0.5 | Extra exploration after losses |

**Tuning Guidance:**
- Higher temperature = more exploration, less exploitation
- Lower minimum = can become very deterministic
- Higher RPE gain = more recovery attempts after losses

#### Gating Thresholds

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `invigoration_threshold` | 0.0-1.0 | 0.75 | Threshold for GO activation |
| `no_go_threshold` | 0.0-1.0 | 0.25 | Threshold for NO-GO inhibition |
| `hold_threshold` | 0.0-1.0 | 0.40 | Threshold for HOLD signal |

**Tuning Guidance:**
- Lower GO threshold = more aggressive entry
- Higher NO-GO threshold = easier to block trades
- Adjust based on false positive/negative analysis

## Best Practices

### 1. Start Conservative
Always begin with conservative settings and gradually increase aggressiveness as you gain confidence in the system's behavior.

### 2. Validate Extensively
Test all configuration changes in backtest before paper trading, and paper trading before live deployment.

### 3. Document Changes
Keep a log of all parameter changes and their observed effects. This helps identify what works and enables quick rollback.

### 4. Monitor Key Metrics
Track these metrics continuously:
- Engagement Index stability
- Mode transition frequency
- Sharpe ratio
- Maximum drawdown
- Win rate and average hold time

### 5. Respect Market Regimes
Different market conditions require different calibrations. Be prepared to adjust when regimes change.

### 6. Use Version Control
Store all configuration files in version control to track changes and enable rollback.

### 7. Test Boundary Conditions
Verify behavior at parameter boundaries:
- What happens at extreme volatility?
- How does the system behave at maximum drawdown?
- Does it recover appropriately?

### 8. Calibrate for Your Capital
Scale risk parameters to your actual account size. A $10K account and $1M account need different settings.

## Troubleshooting

### Problem: Frequent Suspensions

**Symptoms:** Trading suspended too often, missing opportunities

**Causes:**
- `EI_crit` too high
- Volatility thresholds too low
- Risk multipliers too conservative

**Solutions:**
1. Lower `EI_crit` by 0.05
2. Increase `vol_amber` and `vol_red` by 10-20%
3. Increase `activity_mult` values
4. Review energy recovery parameters (`u_e`, `u_l`, `u_p`)

### Problem: Excessive Risk-Taking

**Symptoms:** Drawdowns exceed tolerance, volatility too high

**Causes:**
- Thresholds too high
- Risk multipliers too aggressive
- Insufficient hysteresis

**Solutions:**
1. Lower `vol_amber` and `vol_red` thresholds
2. Reduce `risk_mult.GREEN` and `risk_mult.AMBER`
3. Increase `EI_hysteresis` for more stability
4. Tighten `delta_r_limit` to slow risk changes

### Problem: Erratic Behavior

**Symptoms:** Rapid mode switching, unstable metrics

**Causes:**
- Insufficient hysteresis
- Temperature too high
- Rate limits too loose

**Solutions:**
1. Increase `EI_hysteresis`
2. Add more separation between mode thresholds
3. Reduce `base_temperature` in dopamine controller
4. Tighten `delta_r_limit` for smoother transitions

### Problem: Slow Adaptation

**Symptoms:** System doesn't respond to changing conditions

**Causes:**
- Learning rates too low
- Rate limits too tight
- Temperature too low

**Solutions:**
1. Increase `learning_rate_v`
2. Loosen `delta_r_limit` slightly
3. Increase `base_temperature` for more exploration
4. Review decay rates in dopamine controller

### Problem: Poor Performance

**Symptoms:** Low Sharpe ratio, suboptimal returns

**Process:**
1. Backtest across multiple time periods
2. Compare against benchmark strategies
3. Analyze which component is underperforming
4. Adjust relevant parameters incrementally
5. Re-test and iterate

## Advanced Topics

### Adaptive Calibration

For automated parameter tuning, use the Adaptive Calibrator:

```python
from tradepulse.core.neuro.adaptive_calibrator import (
    AdaptiveCalibrator,
    CalibrationMetrics,
)

# Initialize with current parameters
calibrator = AdaptiveCalibrator(current_params)

# In your trading loop
metrics = CalibrationMetrics(
    sharpe_ratio=...,
    max_drawdown=...,
    # ... other metrics
)

# Get updated parameters
calibrated_params = calibrator.step(metrics)
```

### Custom Profiles

Create custom profiles by editing `scripts/calibrate_controllers.py`:

```python

    "description": "Your custom description",
    "nak": {
        # Your NAK parameters
```

Different assets may require different calibrations. Use symbol-specific overrides:

```yaml

    max_leverage: 3.0
  ETH/USD:
    max_position_size: 100.0
    max_leverage: 4.0
```

## Calibration Constants Module

All parameter ranges, defaults, and invariants are defined in the **calibration_constants** module:

**Location**: `core/neuro/calibration_constants.py`

This module serves as the single source of truth for:
- Parameter range definitions (min/max bounds)
- Default values for all controllers
- Invariant relationships (e.g., low < high)
- Validation functions

**Key Features:**
- **Zero Magic Numbers**: All hardcoded values eliminated
- **Centralized Validation**: One function validates all controller types
- **Clear Error Messages**: Violations specify parameter, value, and constraint
- **Type Safety**: Frozen dataclasses prevent accidental modification

**Example Usage:**
```python
from core.neuro.calibration_constants import (
    NAKParameterRanges,
    validate_parameter_invariants,
)

# Get parameter ranges
ranges = NAKParameterRanges()
print(f"EI range: {ranges.EI_RANGE}")
print(f"EI_low default: {ranges.EI_LOW_DEFAULT}")

# Validate parameters
params = {"EI_low": 0.35, "EI_high": 0.65, "EI_crit": 0.15}
is_valid, errors = validate_parameter_invariants("nak", params)
if not is_valid:
    for error in errors:
        print(f"Error: {error}")
```

**Extending the System:**

To add a new controller or parameter:

1. Add parameter ranges in a new dataclass (e.g., `MyControllerParameterRanges`)
2. Add validation function (e.g., `_validate_my_controller_invariants`)
3. Register in `validate_parameter_invariants()` function
4. Add calibration profiles in `scripts/calibrate_controllers.py`
5. Add tests in `tests/core/neuro/test_calibration_constants.py`
