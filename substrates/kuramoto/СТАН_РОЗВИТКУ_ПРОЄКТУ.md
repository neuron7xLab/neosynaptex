## 🧭 План доведення архітектури та структури до готовності (до v1.0)

- **Опублікувати всі решта ADR** для крос-модульних рішень, щоб 100% архітектурних змін були трасовані.
- **Заблокувати контракти модулів та guardrails залежностей** (SemVer-перевірка + import lint) через CI-гейт.
- **Підтвердити топологію та observability-базові значення** проти production SLO через failover drills та telemetry snapshots.
- **Провести Architecture Readiness Review** (ARB) з беклогом remediation та іменованими власниками; усі P0 мають бути прийняті.
- **Заморозити діаграми та схеми** у `docs/architecture/assets/` та `docs/schemas/` перед першим release-tag.

**Сигнал приймання:** Sign-off ARB + зелений architecture CI gate, діаграми/схеми відповідають розгорнутій топології.

## 🚀 Точки росту та безпечні дії вдосконалення (best practices)

- **Архітектурна цілісність (SWEBOK/ADR discipline)**  
  - Закріпити канонічний шлях `tradepulse.*`, заблокувати legacy-шим імпорти через статичний import-lint у CI.  
  - Кожна зміна між доменами → ADR + traceability у `docs/requirements/traceability_matrix.md`.
- **Надійність і SRE-контроль**  
  - Для кожного сервісу: SLO/SLI (latency p95/p99, error budget burn-rate), автогенерація дашбордів і алертів.  
  - Регулярні kill-switch/failover дрилі з фіксацією Free Energy (`runtime/thermo_controller.py`) і rollback SLA ≤30s.
- **Тестова зрілість та якість**  
  - Пріоритезувати property-based покриття execution/backtest; mutation kill-rate ≥90% як мердж-гейт.  
  - Perf regression guardrail: бенчмарки `tests/performance/` у pre-merge для гарячих шляхів (<5ms order latency, design target).
- **Supply Chain Security (SLSA/SBOM)**  
  - Підписаний SBOM (CycloneDX + sigstore), 7-денний SLA на критичні CVE з release-блокером.  
  - Secret scanning + provenance (attestations) увімкнені для Docker/Python артефактів.
- **Дані, конфігурація та ізоляція середовищ**  
  - Строгий поділ prod/stage/dev через Hydra schema validation + drift detection.  
  - Репліковані seed-и/датасети для детермінізму; data contracts для інтерфейсів ingestion/feature store.

Ці growth-points опираються на провідні практики (SRE, SLSA, NIST SP 800-53 design alignment) і мають бути виконані під контролем dual-approval у TACL, щоб тримати зміни у межах monotonic free-energy descent.
