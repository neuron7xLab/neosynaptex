# TradePulse Risk Guardian — Product Pain & Solution

## 🎯 Target User

**Manual / Quasi-Algo Trader** — трейдер, що торгує вручну або з напів-автоматизованими системами, але не має системного контролю ризику.

**Характеристики:**
- Торгує криптовалюти, Forex або CFD
- Має стратегію (технічний аналіз, сигнали, або власні моделі)
- НЕ має вбудованого автоматичного risk management
- Регулярно втрачає 10-30%+ депозиту через великі просадки

---

## 💔 The Pain (Біль)

> **"Немає системного risk-control → зліт депо"**

### Симптоми:
1. **Катастрофічні просадки** — один невдалий день/тиждень знищує місяці прибутку
2. **Емоційні рішення** — під час просадки трейдер або панікує, або "усереднює" позицію, погіршуючи ситуацію
3. **Відсутність kill-switch** — немає автоматичного механізму зупинки торгівлі при критичних втратах
4. **Невідомий ризик** — трейдер не знає свій реальний max drawdown заздалегідь

### Реальні цифри (типовий сценарій):
- Трейдер робить +15% за місяць
- Один поганий день: -25% (без контролю ризику)
- Результат: -10% за місяць замість +15%
- **Втрачено: 25% потенційного прибутку**

---

## 💡 The Solution (Рішення)

> **TradePulse Risk Guardian** — автоматичний контроль ризику, що працює поверх будь-якої торгової стратегії.

### Product Statement

**"TradePulse Risk Guardian обмежує максимальну просадку вашого портфеля до заданого рівня, автоматично зупиняючи торгівлю при критичних втратах і переводячи в safe-mode при попереджувальних рівнях."**

### Цінність у грошах:

| Сценарій | Без Risk Guardian | З Risk Guardian | Економія |
|----------|-------------------|-----------------|----------|
| Катастрофічна просадка | -25% за день | -10% (cap) | **15%** депозиту |
| Серія поганих трейдів | -40% за тиждень | -15% (kill-switch) | **25%** депозиту |
| Емоційне переторгування | Нескінченне | Заблоковано | **Весь залишок** |

### Як це працює:

1. **Daily Loss Cap** — автоматичне обмеження денних втрат (наприклад, -5%)
2. **Drawdown Kill-Switch** — зупинка торгівлі при досягненні порогу (наприклад, -10%)
3. **Safe Mode** — автоматичне зменшення розміру позицій при попереджувальних рівнях
4. **Position Limits** — обмеження максимального розміру окремих позицій

## 🔧 Technical Implementation

### CLI Interface:
```bash
# Запуск Risk Guardian з конфігурацією
tp risk-guardian run --config=config.yaml

# Симуляція на історичних даних
tp risk-guardian simulate --history=trades.csv --config=config.yaml

# Звіт: скільки збитків відсікли
tp risk-guardian report --history=trades.csv
```

### Ключові метрики:
- **Saved Capital** — скільки грошей збережено завдяки kill-switch
- **Drawdown Reduction** — на скільки зменшена максимальна просадка
- **Blocked Catastrophic Trades** — кількість відсічених катастрофічних угод
- **Sharpe Delta** — покращення risk-adjusted return

## ⚠️ Limitations

1. **Trade-off: Return vs Risk** — обмеження ризику може знизити абсолютний прибуток
2. **Not a Strategy** — Risk Guardian не генерує торгові сигнали, лише контролює ризик
3. **Gaps Risk** — не захищає від gap-ів (раптові стрибки ціни на закритому ринку)
4. **Configuration Sensitivity** — ефективність залежить від правильного налаштування порогів

```bash

pip install tradepulse[risk_guardian]

# Швидкий тест на sample даних
tp risk-guardian simulate --history=sample.csv --daily-loss-limit=5 --max-drawdown=10

# Повний звіт
tp risk-guardian report --history=your_trades.csv --output=report.md
```
