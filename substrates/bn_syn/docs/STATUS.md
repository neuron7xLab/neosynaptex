# Project Status

This project is research-grade / pre-production. No battle usage claimed.

## Machine-readable release readiness

The live repository readiness verdict is **not** declared manually in this page.
The only machine-readable release readiness source of truth is the execution-backed
report generated by `python -m scripts.release_readiness` and serialized to
`artifacts/release_readiness.json`.

The readiness state vocabulary is fixed to:

- `blocked`
- `advisory`
- `ready`

Interpretation:

- `blocked`: at least one blocking subsystem failed, or zero execution-backed
  checks passed.
- `advisory`: all blocking subsystems passed, but at least one non-blocking
  advisory condition remains.
- `ready`: all subsystems passed and at least one execution-backed check passed.

## Scope of the statement

- BN-Syn is developed and validated as a deterministic research system.
- CI, tests, and documentation demonstrate reproducibility and governance controls.
- Production adoption, external deployment, and battle-tested usage are not claimed in this repository unless accompanied by explicit external evidence artifacts.
- Readiness claims must cite the current `artifacts/release_readiness.json` report rather than a static sentence in this document.
